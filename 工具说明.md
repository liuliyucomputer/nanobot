# nanobot 可用工具说明

## 文件操作

### read_file - 读取文件
读取文件内容。
```
read_file(path: str) -> str
```
**示例**: 读取配置文件
```bash
./venv/bin/nanobot agent -m "读取文件 /root/.nanobot/config.json 的内容"
```

### write_file - 写入文件
向文件写入内容（如果需要会自动创建父目录）。
```
write_file(path: str, content: str) -> str
```
**示例**: 创建新的 Python 脚本
```bash
./venv/bin/nanobot agent -m "创建一个 hello.py 文件，内容是 print('你好世界')"
```

### edit_file - 编辑文件
通过替换特定文本来编辑文件。
```
edit_file(path: str, old_text: str, new_text: str) -> str
```
**示例**: 修改配置文件
```bash
./venv/bin/nanobot agent -m "在 config.json 中将模型改为 gpt-4"
```

### list_dir - 列出目录
列出目录内容。
```
list_dir(path: str) -> str
```
**示例**: 查看当前目录文件
```bash
./venv/bin/nanobot agent -m "列出当前目录的所有文件"
```

## Shell 执行

### exec - 执行命令
执行 shell 命令并返回输出。
```
exec(command: str, working_dir: str = None) -> str
```
**示例**: 检查系统状态
```bash
./venv/bin/nanobot agent -m "执行命令 df -h 查看磁盘使用情况"
```

## 安全注意事项

⚠️ **重要提醒**:
- nanobot 可以执行系统命令，请谨慎使用
- 文件操作会覆盖现有文件，建议先备份
- 在生产环境中使用前先在测试环境验证

## 工具调用原理

nanobot 通过以下方式使用工具：
1. **智能识别**: 根据你的请求自动选择合适的工具
2. **参数解析**: 自动提取和格式化所需参数
3. **结果处理**: 将工具输出整合到对话中
4. **错误处理**: 提供清晰的错误信息和解决建议

## 扩展功能

除了基础工具，nanobot 还支持：
- **网络请求**: 访问 API 和网页
- **数据库操作**: 连接和查询数据库
- **定时任务**: 定期执行操作
- **多模态**: 处理图像、音频等（需要相应模型）
